# Agent Loop Protocol

This repository uses a GitHub PR handoff loop so each major phase is auditable before work continues.

## Phase Gate (Required After Every Major Phase)

1. Run tests and validators relevant to the phase.
2. Write `reports/agent_handoffs/latest_handoff.md` and `reports/agent_handoffs/latest_handoff.json`.
3. Commit a checkpoint on the working branch.
4. Push the branch to GitHub.
5. Open or update the phase PR.
6. Paste PR body generated by `scripts/dev.cmd prepare-pr-handoff`.
7. Stop and wait for audit feedback before starting the next major phase.

## PR Convention

- PR title format: `[Cursor Phase] <short phase name>`
- Keep one major phase per PR whenever possible.
- Ensure each PR has a fresh handoff report and generated PR handoff body.

## Standard Commands

- Write handoff reports:
  - `scripts/dev.cmd write-agent-handoff --phase "<phase>" --goal "<goal>"`
- Prepare PR body:
  - `scripts/dev.cmd prepare-pr-handoff`

## Audit Expectations

Every PR should contain enough context for an independent audit to verify:

- The diff matches the phase goal.
- Tests and validators were executed and outcomes were captured.
- Constraints were followed.
- Risks and unresolved decisions are explicitly documented.
